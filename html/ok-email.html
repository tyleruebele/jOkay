ok-email

A pure JavaScript (no dependencies) solution to make HTML email inputs self-validating

Usage

  1. Include ok-email.js anywhere in your page, it runs on load.
  2. Add CSS class 'js-ok-email' to all email inputs you want validated.
  3. Optionally include ok-email.css (or its contents) in your styles.

Validating Lists

If an input or textarea is meant to accept a list of email addresses, such can be validated by adding class 'js-ok-email-list' to the element.

Stricter / Looser Usage

By default, email domains are validated in a semi-strict internet format. A short list of TLDs is checked, and any two letter TLD is assumed to be a valid country code. Emails can be validated against a looser or stricter domain format. To skip TLD validation and accept local (one part) domains, apply CSS class 'js-ok-email-looser' to the input. Likewise, to enforce known TLDs, apply css class 'js-ok-email-stricter' to the input. When using stricter testing, it is important to frequently update the list of TLDs.


Styling Errors

Inputs with invalid emails can be styled by applying styles to the error class. By default this class is 'js-ok-email-error'.
