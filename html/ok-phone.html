ok-phone

A pure JavaScript (no dependencies) solution to make HTML phone inputs self-validating

Usage

  1. Include ok-phone.js anywhere in your page, it runs on load.
  2. Add CSS class 'js-ok-phone' to all phone inputs you want validated.
  3. Optionally include ok-phone.css (or its contents) in your styles.

Stricter / Looser Usage

By default, phone numbers are validated in a loose NANPA format. A short list of rules is checked. Phones can be validated against a looser or stricter format. To accept letters for phonewords, add CSS class 'js-ok-phone-wordy' to the input. Likewise, to enforce known area codes, apply css class 'js-ok-phone-strict' to the input. When using strict testing, it is important to frequently update the list of area codes.


Styling Errors

Inputs with invalid phones can be styled by applying styles to the error class. By default this class is 'js-ok-phone-error'.
