Demo for validate-email.js

validate-email.js is a pure JavaScript (no dependencies) solution to make HTML email inputs self-validating.

Basic Usage

The basic email validation functionality can be added simply by including validate-email.js anywhere in your document and adding CSS class 'js-validate-email' or type 'email' to all inputs you want validating. It works like this:


Stricter / Looser Usage

By default, email domains are validated in a semi-strict internet format. A short list of TLDs is checked, and any two letter TLD is assumed to be a valid country code. Emails can be validated against a looser or stricter domain format. To skip TLD validation and accept local (one part) domains, apply CSS class 'js-validate-email-looser' to the input. Likewise, to enforce known TLDs, apply css class 'js-validate-email-stricter' to the input. When using stricter testing, it is important to frequently update the list of TLDs.


Styling Errors

Inputs with invalid emails can be styled by applying styles to the error class. By default this class is 'js-validate-email-error'.
